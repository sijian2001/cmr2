# CRM システム

Ruby on Rails 8ベースの包括的な顧客関係管理 (CRM) システムです。

## 概要

このプロジェクトは、企業の顧客管理、製品管理、店舗管理、店員管理を統合的に行うためのWebアプリケーションです。Devise認証システムを使用し、包括的なテストスイートを備えています。

## 主要機能

### 認証システム (Devise)
- ユーザー登録・ログイン・ログアウト
- セッション管理
- パスワードリセット機能
- 認証必須ページ保護

### 顧客管理
- 顧客登録・編集・削除・詳細表示
- 顧客一覧表示とページネーション
- 名前・メール・電話番号での検索機能
- バリデーション（名前・メール必須、メール形式チェック）

### 製品管理
- 製品登録・編集・削除・詳細表示
- 製品一覧表示とページネーション
- 名前・説明・SKUでの検索機能
- 在庫管理機能
- バリデーション（名前・価格必須、価格正数チェック）

### 店舗管理
- 店舗登録・編集・削除・詳細表示
- 店舗一覧表示とページネーション
- 名前・住所・ステータスでの検索機能
- ステータス管理（営業中・閉店・準備中）
- バリデーション（名前・住所必須）

### 店員管理
- 店員登録・編集・削除・詳細表示
- 店員一覧表示とページネーション
- 名前・メール・従業員IDでの検索機能
- 店舗との関連付け
- バリデーション（名前・メール・従業員ID必須）

## 技術仕様

- **フレームワーク**: Ruby on Rails 8.0.3
- **認証**: Devise
- **データベース**: MySQL (開発・本番) / SQLite3 (テスト)
- **フロントエンド**: Hotwire (Turbo + Stimulus), Importmap
- **テスト**: Minitest, Playwright (E2E)
- **コード品質**: RuboCop, Brakeman
- **デプロイ**: Kamal

## セットアップ手順

### 1. 依存関係のインストール
```bash
bundle install
```

### 2. データベースの設定
```bash
bin/rails db:create
bin/rails db:migrate
bin/rails db:seed
```

### 3. 開発サーバーの起動
```bash
bin/dev
```

アプリケーションは http://localhost:3000 でアクセス可能です。

## 開発コマンド

### サーバー・開発
- `bin/dev` - 開発サーバーを起動（Railsサーバー）
- `bin/rails server` - サーバーを起動する別の方法
- `bin/rails console` - デバッグ・テスト用のRailsコンソールを開く

### テスト・品質管理
- `bin/rails test` - テストスイートを実行
- `bin/rails test:e2e` - Playwright E2Eテストを実行
- `bin/rubocop` - Rubyのリンティング・スタイルチェックを実行
- `bin/brakeman` - セキュリティ脆弱性スキャンを実行

### データベース
- `bin/rails db:create` - データベースを作成
- `bin/rails db:migrate` - データベースマイグレーションを実行
- `bin/rails db:seed` - シードデータを読み込み
- `bin/rails db:reset` - データベースを削除、作成、マイグレーション、シードを実行

## データベース設定

### 開発・本番環境 (MySQL)
```yaml
データベース: mysql2
ホスト: localhost
ポート: 3306
データベース名: crm2 (開発) / crm2_production (本番)
ユーザー名: crm2user
パスワード: 1234
```

### テスト環境 (SQLite3)
```yaml
データベース: sqlite3
ファイル: storage/test.sqlite3
```

## テストスイート

包括的なテストスイートを備えており、以下をカバーしています：

### モデルテスト
- バリデーションテスト
- 検索機能テスト
- 関連付けテスト

### コントローラーテスト
- CRUD操作テスト
- 認証機能テスト
- 検索機能テスト

### E2Eテスト (Playwright)
- ブラウザベースのエンドツーエンドテスト
- 顧客管理のCRUD操作テスト
- ページアクセステスト
- CI/CD統合

### フィクスチャ
- 現実的なテストデータ（日本語対応）
- Devise認証に対応したユーザーフィクスチャ

テストの実行:
```bash
# 全テストを実行
bin/rails test

# E2Eテストを実行
bin/rails test:e2e
```

E2Eテストの詳細なセットアップ手順は [docs/PLAYWRIGHT_SETUP.md](docs/PLAYWRIGHT_SETUP.md) を参照してください。
