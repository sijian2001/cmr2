# CRMシステム設計書

## 1. システム概要

### 1.1 目的
Ruby on Rails 8を基盤とした包括的な顧客関係管理（CRM）システムの構築。企業の顧客、製品、店舗、店員の統合管理を行う。

### 1.2 対象ユーザー
- 企業の営業担当者
- 店舗管理者
- システム管理者

### 1.3 主要目標
- 顧客情報の一元管理
- 製品・在庫管理の効率化
- 店舗・店員管理の統合
- セキュアな認証システム
- 高度な検索・フィルタリング機能

## 2. システムアーキテクチャ

### 2.1 技術スタック
```
フレームワーク: Ruby on Rails 8.0.3
認証システム: Devise
データベース: MySQL (開発・本番) / SQLite3 (テスト)
フロントエンド: Hotwire (Turbo + Stimulus)
アセット管理: Importmap
テストフレームワーク: Minitest
コード品質: RuboCop, Brakeman
デプロイ: Kamal
```

### 2.2 MVCアーキテクチャ
```
Model層: データ管理、バリデーション、ビジネスロジック
View層: ERBテンプレート、レスポンシブUI
Controller層: リクエスト処理、認証、データフロー制御
```

### 2.3 データベース設計

#### 2.3.1 テーブル構成
```
users (ユーザー認証)
├── id (Primary Key)
├── email (認証用メールアドレス)
├── encrypted_password (暗号化パスワード)
└── 認証関連メタデータ

customers (顧客管理)
├── id (Primary Key)
├── name (顧客名) [必須]
├── email (メールアドレス) [必須、ユニーク]
├── phone (電話番号)
├── address (住所)
└── タイムスタンプ

products (製品管理)
├── id (Primary Key)
├── name (製品名) [必須]
├── description (説明)
├── price (価格) [必須、正数]
├── sku (商品コード) [ユニーク]
├── stock_quantity (在庫数)
└── タイムスタンプ

stores (店舗管理)
├── id (Primary Key)
├── name (店舗名) [必須]
├── address (住所) [必須]
├── phone (電話番号)
├── status (営業ステータス)
└── タイムスタンプ

staffs (店員管理)
├── id (Primary Key)
├── name (店員名) [必須]
├── email (メールアドレス) [必須]
├── phone (電話番号)
├── employee_id (従業員ID) [必須、ユニーク]
├── store_id (所属店舗) [外部キー]
└── タイムスタンプ
```

#### 2.3.2 リレーション設計
```
Store ←→ Staff (1対多の関係)
- 1つの店舗に複数の店員が所属
- 店員は必ず1つの店舗に所属
```

### 2.4 認証・セキュリティ設計

#### 2.4.1 認証システム (Devise)
```
機能:
- ユーザー登録・ログイン・ログアウト
- パスワードリセット
- セッション管理
- 認証必須ページ保護

セキュリティ機能:
- CSRF保護
- パスワード暗号化
- セッションタイムアウト
- モダンブラウザ要求
```

#### 2.4.2 アクセス制御
```
全ページ: 認証必須 (before_action :authenticate_user!)
ホームページ: 認証後のランディングページ
各CRUDページ: 認証ユーザーのみアクセス可能
```
